makef.io

Yesterdays technology -- today!

Use Makefiles to run your software project's life cycle. makef.io's goal is to provide common targets through most programming languages and frameworks.  make test should work in python, java, ruby or javascript.

You may already have a framework like Django, Ruby on Rails, or gradle, but Makefiles will give you a standard interface to interact with with your projects. makef.ios "just work."

make is on every developer's machine and isn't going away any time soon.

Including makef.io

For the initiated who already use Makefiles, you can include makef.ios by including this at the top of your Makefile.

$(shell test -f Makef.io || curl -o Makef.io makef.io/python)
include Makef.io

test -f Makef.io will check to see if a Makef.io file exists, and curl will download the file if it does not. Notice the /python. This will download a stock Makefile for python. It includes test, clean, compile targets. You may mix multiple Makef.ios together by continuing the slashes. makef.io/python/travisci will include python targets and a target to build a .travis.yml build script for the project.

What do you get?

This service is a proxy for github.com/trevorgrayson/makefiles. Keep an eye out for new supported technologies. New features will help you compose many Makefiles for your project.

  • makef.io/docker will add targets for only docker.
  • makef.io/docker/python will add targets for docker and python. Later makef.ios have greater priority.
  • makef.io/python?PYTHON=python3 will set the PYTHON Environment variable
